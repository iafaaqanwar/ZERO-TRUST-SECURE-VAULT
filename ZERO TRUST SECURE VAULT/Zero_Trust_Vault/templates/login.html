Zero Trust Storage

Local Directories · JSON Logs

Security Enforcement Role:

Zero Trust Data Lifecycle

1
Step 1

Identity Ingestion & Authentication

User registers with a verified email domain (@zerotrustvault.com). Credentials are encrypted using salted PBKDF2-SHA256 hashes via werkzeug.security before persistence in local storage.

2
Step 2

Cryptographic Hashing (Integrity Anchor)

The uploaded file is loaded strictly into volatile memory. A secure SHA-256 fingerprint hash is computed instantly to establish a baseline for non-repudiation and tampering checks.

3
Step 3

Symmetric Multi-Layer Encryption

The raw file bytes are encrypted in memory via AES-128-CBC (Fernet). Plaintext data is immediately wiped from memory buffers, ensuring no plaintext ever touches the storage disk.

4
Step 4

Isolated Storage & Auditing

The resulting .enc asset is placed into the owner's isolated folder. Simultaneously, a permanent record of the file event is appended to the tamper-evident audit_logs.json ledger.
